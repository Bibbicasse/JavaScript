l'ordre d'exécution du code par le navigateur

Par défaut, un navigateur va lire et exécuter le code dans l'ordre de son écriture

Lorsque le navigateur arrive à un script, il va stopper le traitement du reste du HTML jusqu'à ce que le code JavaScript soit chargé dans la page et exécuté

Résumons ce qui ce passe dans ce cas

  1. Le navigateur commance à analyser (ou à traiter) le code HTML
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et téléchargé (si il est en externe) puis exécuté. Durant tout ce temps l'analyseur du navigateur bloque l'affichage du HTML ce qui peut ralentir, si le script est long, l'affichage de la page ;
  4. Dès que le JS est exécuté, le contenu HTML est affiché

ce soucis d'attente est résolu grâce au téléchargement asynchrone des données avec l'attribut async ou defer dans le cas où l'on fait appel à des fichier externe

Le point virgule

Le point virgule ; est utilisé pour signifier la fin d'une instruction (une variable, une condition, une boucle etc.).
